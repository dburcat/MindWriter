MindWriter API

REST API for the MindWriter notes manager. All endpoints return JSON.
Base URL: http://localhost:8000

Overview
The API reads and writes notes from ~/.notes/notes/ and datasets from ~/.notes/datasets/. Override the root with the NOTES_DIR environment variable.
Starting the server
# Install dependencies pip install flask flask-cors # Run (default port 8000) python3 mindwriter_api.py # Custom port or notes directory PORT=9000 python3 mindwriter_api.py NOTES_DIR=/my/notes python3 mindwriter_api.py # Debug mode (no auto-shutdown, no browser open) DEBUG=true python3 mindwriter_api.py
Authentication
All endpoints except /, /health, /ping, and /api/auth/key require an API key. A key is auto-generated on first run and saved to ~/.notes/.api_key.
Passing the key
# Via header (recommended) curl -H "X-API-Key: YOUR_KEY" http://localhost:8000/api/notes # Via query param curl http://localhost:8000/api/notes?api_key=YOUR_KEY
Rate limiting: 120 requests per 60-second window by default. Override with RATE_LIMIT_MAX and RATE_LIMIT_WINDOW env vars. Exceeding the limit returns 429 Too Many Requests.

Localhost only: The server binds to 127.0.0.1 and rejects all non-loopback connections with 403 Forbidden.
Error Codes
CodeMeaning
400Bad request — missing or invalid parameter
401Unauthorized — API key missing or wrong
403Forbidden — connection not from localhost
404Not found — note or dataset does not exist
409Conflict — duplicate title or filename
429Too many requests — rate limit exceeded
500Internal error — file system operation failed
// All errors return the same shape: { "error": "Human-readable description" }
System
GET /health Server health check · public

Returns server status, resolved notes directory, and note count. Does not require authentication.

Response 200
{ "status": "ok", "notes_root": "/home/user/.notes", "notes_dir": "/home/user/.notes/notes", "notes_dir_exists": true, "note_count": 42 }
GET /api/auth/key Retrieve API key · public · localhost only

Returns the current API key. No authentication required. Safe because the server only accepts localhost connections — only someone on the same machine can call this.

Response 200
{ "api_key": "your-key-here" }
GET /ping Keepalive heartbeat · public

Called by the UI every 5 seconds. Resets the auto-shutdown watchdog timer. The server exits ~15 seconds after pings stop arriving (configurable via SHUTDOWN_TIMEOUT).

Response 200
{ "ok": true }

Returns all notes with metadata (no body content). Supports filtering and sorting.

GET /api/notes/:id Get one note with body

Returns full note including body content. :id can be a numeric index or the exact filename.

200 OK404 Not Found401 Unauthorized
GET /api/notes/search Search notes by keyword

Searches filename, all YAML header fields, and body content. OR logic — a note is included if any keyword matches. Repeat ?q= for multiple keywords, or use comma-separated values.

Query Parameters
NameTypeRequiredDescription
qstringrequiredKeyword to search. Repeat for multiple: ?q=python&q=flask
Response 200
{ "query": ["python", "flask"], "total": 3, "notes": [ { "id": 1, "title": "flask setup", "matched_keywords": ["flask"], "match_report": { "flask": { "locations": ["header:title", "body"], "snippets": ["…Install Flask with pip install flask…"] } } } ] }
200 OK400 Bad Request401 Unauthorized
POST /api/notes Create a note

Creates a new note file. The filename is derived from the title (slugified). If a file with that name already exists a numeric suffix is appended.

Request Body (JSON)
FieldTypeRequiredDescription
titlestringrequiredNote title. Stored lowercase.
bodystringoptionalNote content (plain text / Markdown)
authorstringoptionalAuthor name. Stored lowercase.
tagsstring|arrayoptionalComma-separated string or JSON array
prioritystringoptionale.g. high, low, 1–5
Response 201
{ "message": "Note created.", "id": 7, "filename": "my-note.md", "title": "my note" }
201 Created400 Bad Request401 Unauthorized

Upload a .md, .txt, or .note file. If the file has no YAML front matter, a header is automatically prepended with the filename as the title.

POST /api/datasets/upload Upload a dataset file from browser

Accepts a multipart file upload. Dataset titles must be unique (409 on duplicate). Large uploads can be cancelled mid-flight via DELETE /api/upload/cancel/:id.

Request (multipart/form-data)
FieldTypeRequiredDescription
filefilerequired.csv or .json file
upload_idstringoptionalClient-generated UUID used to cancel this upload
title, description, author, tags, source_url, license, prioritystringoptionalSidecar metadata fields
Response 201
{ "message": "Dataset uploaded.", "filename": "sales.csv", "rows": 1500, "columns": 8, "parse_error": null }
If the upload is cancelled, the response is 200 with {"cancelled": true} rather than 201.

POST /api/datasets/:n/columns Add columns to a dataset

Adds new columns to an existing dataset. Existing rows receive the specified default value for each new column. Columns that already exist are silently skipped.

Uploads
DELETE /api/upload/cancel/:upload_id Cancel an in-progress upload

Signals a running dataset upload (or reupload) to abort. The upload endpoint polls for this signal between chunks and during row parsing. The temp file is deleted on cancellation. upload_id must match the value sent in the original upload request.

Response 200
{ "message": "Cancel signal sent.", "upload_id": "abc123…" }
200 OK401 Unauthorized
